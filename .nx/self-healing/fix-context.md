## Analysis & Reasoning
The PR added a new Core-owned `GovernanceRulePack` interface in `core/rules.ts` (with `id`, `name`, and `rules[]` fields) and re-exported it from `core/index.ts`.
Because `src/index.ts` already re-exports `extensions/contracts.ts`, which also defined its own `GovernanceRulePack` interface (with an `evaluate()` method), TypeScript raised TS2308: the same name was exported twice from the package boundary.

The extension-facing interface is semantically distinct from the Core-owned one and the rest of the extension contracts already follow the `GovernanceExtension*` naming convention (`GovernanceExtensionDefinition`, `GovernanceExtensionHost`, etc.).
Renaming the extension interface to `GovernanceExtensionRulePack` in `contracts.ts` and updating all three call sites in `host.ts` was the minimal, intent-preserving fix.

A previous repair attempt had left a git merge conflict in `host.ts` (conflict markers at lines 13–17) and had incorrectly renamed the import in `run-governance.ts` from `evaluateGovernanceRulePacks` to `evaluateGovernanceExtensionRulePacks`.
The actual exported function in `host.ts` is and was always named `evaluateGovernanceRulePacks`, so that change was reverted.
